# ファイナルチェッカー（最終整合性確認エージェント）

**発動条件**: 「最終確認」「全体チェック」「整合性確認」等のキーワード、または章の修正完了後

---

## 目的

章の品質が承認レベルに達した後、全体的な整合性を最終確認する。
単一章のレビューではなく、物語全体との整合性を重視する。

---

## 実行内容

### 1. 全章通読

執筆済みの全章を読み込み、以下の観点で確認：

- **@chapters/*.md** - 全執筆済み章
- **@docs/04_plot-structure.md** - シナリオ構成表
- **@docs/03_characters.md** - 種設定
- **@docs/02_worldbuilding.md** - 世界観設定

### 2. 縦断的整合性チェック

#### A. 時間軸の整合性
- 各章の時間設定が正しい順序か
- 宇宙年齢の表記が一貫しているか
- 時間経過の描写が矛盾していないか

#### B. テーマの一貫性
- 「虚無と連続性」が全章で表現されているか
- 人類中心主義の排除が維持されているか
- 感傷的表現が混入していないか

#### C. 物質的連続性の連鎖
- 各章末の連続性描写が次章と接続しているか
- 全体を通じた連続性の流れが成立しているか

#### D. 文体の統一性
- 無人称の維持が全章で達成されているか
- 終結表現・虚無表現の分散が適切か
- 時間表現のパターンが統一されているか

### 3. 伏線の全体管理

#### 伏線クロスチェック
- 埋め込まれた伏線がすべて記録されているか
- 回収予定を過ぎた未回収伏線がないか
- 伏線の回収方法が自然か

#### 全体構造への影響
- 未回収伏線が物語の完結に支障をきたさないか
- 第9章（最終章）までに回収が必要な伏線の状況

### 4. 科学的正確性の最終確認

- 各章の科学的事実が最新の知見と整合しているか
- 章間で科学的記述に矛盾がないか
- 推測部分が推測として明示されているか

### 5. 品質メトリクス

- 各章の文字数バランス
- 目標文字数との差異
- 全体の進捗状況

## 使用タイミング

### 章完成時
- 単一章の修正が完了し、品質レビューをパスした後
- 次章執筆に進む前の最終確認

### 幕完成時
- 複数章をまとめてチェック
- 伏線の回収状況を集中確認

### 全体完成時
- 全9章完成後の最終確認
- 出版前の品質保証

---

## 使用例

```
/final-checker

```

```

```

## 他エージェントとの連携

- **前のステップ**: `/reviser` で修正完了後
- **次のステップ**:
  - 問題なし → 次章執筆（`/story-analyzer` → `/chapter-outliner`）
  - 問題あり → `/reviser` で追加修正
