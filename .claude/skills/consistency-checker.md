# 整合性チェッカー

**発動条件**: 章の執筆完了後、または「整合性チェック」「矛盾確認」等のキーワード

---

## 実行内容

指定された章（または最新の執筆章）の内容を以下のドキュメントと照合し、矛盾を報告します。

### 1. シナリオ構成表との整合性（@docs/04_plot-structure.md）

確認項目：
- [ ] 該当章の「目的」を達成しているか
- [ ] シーン構成が計画通りか
- [ ] 伏線の【埋込】が記録されているか
- [ ] 回収予定の伏線を【回収】したか
- [ ] 前章からの接続が自然か
- [ ] 次章への接続ポイントが明確か

### 2. 世界観設定との整合性（@docs/02_worldbuilding.md）

確認項目：
- [ ] 世界のルールに違反していないか
- [ ] 技術/魔法の使用が設定範囲内か
- [ ] 地理・距離・移動時間が正確か
- [ ] 社会・文化の描写が設定と一致しているか
- [ ] 時系列に矛盾がないか

### 3. キャラクター設定との整合性（@docs/03_characters.md）

確認項目：
- [ ] キャラクターの言動が性格設定と一致しているか
- [ ] 口調・話し方が一貫しているか
- [ ] 知らないはずの情報を知っていないか
- [ ] 動機と行動が一致しているか
- [ ] 関係性の描写が設定と矛盾していないか

## 使用例

```
/consistency-checker 第3章
```

または章を執筆完了した際に自動的に実行を提案。
