# 章サマリー生成

**発動条件**: 「章サマリー」「章まとめ」「進捗更新」等のキーワード、または章執筆完了時

---

## 実行内容

執筆完了した章から情報を抽出し、@docs/04_plot-structure.md の更新を支援します。

### 抽出項目

1. **シーン要約**
   - 各シーンを1行で要約
   - 時間・場所の変化を記録

2. **登場キャラクター**
   - 章に登場した全キャラクター
   - 新規登場キャラクターの識別

3. **伏線管理**
   - 埋め込んだ伏線の検出
   - 回収した伏線の確認

4. **接続情報**
   - 前章からの接続
   - 次章への接続ポイント

5. **メトリクス**
   - 文字数カウント
   - 推定読了時間

---

### @docs/04_plot-structure.md 更新提案

以下の内容でシナリオ構成表を更新することを推奨:

```markdown
### 第X章：[章タイトル]

**状態**: ✅完了

| 要素 | 内容 |
|------|------|
| **時間** | 20XX年X月X日 XX:XX 〜 XX:XX |
| **場所** | [場所1] → [場所2] |
| **POV** | [視点人物] |
| **目的** | [達成した目的] |
| **文字数** | 目標: X,000字 / 実績: X,XXX字 |

#### シーン構成
1. [シーン1]
2. [シーン2]
3. [シーン3]

#### 伏線
| タイプ | 内容 | 回収予定 | 状態 |
|--------|------|----------|------|
| 【埋込】 | [伏線] | 第Y章 | ⬜未回収 |
| 【回収】 | [伏線] | - | ✅回収済 |
```

### 次のアクション

```

## 使用例

```
/chapter-summary 第3章
```

または章執筆完了時に自動的に実行を提案。
